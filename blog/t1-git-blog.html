Version Control

At it's core Git is a Version Control System. So what does Version Control mean?
Version Control is a method that allows you to create a new version every time you modify your code (instead of overwriting the copy you're currently working on). GitHub uses version control to allow more than one programmer to work on the same piece of code without overwriting each other. It makes sure that whenever a piece of the code is modified by any contributor, a new version of the whole code is created, it also shows which part of the code was modified and by whom.
