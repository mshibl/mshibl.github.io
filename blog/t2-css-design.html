CSS positioning in 4 simple examples

August 10, 2015

A web-page is very similar to a normal building. This is an analogy that helped me understand the basics of web design. A building has an inner structure (lobby, rooms, hallways etc.), in the case of a web-page the inner structure is the HTML, the rooms are the different elements that makeup an HTML document (header, body, footer etc.). Once a building structure is in place, the decoration then takes place (wall paint, curtains, floorings etc.), in the case of a web-page the decoration is done using CSS, using CSS you can add the background colors, you can choose the type & size of the font , you can position different elements and much more!

Learning CSS is easy, the syntax is easy to grasp and the tools to learn is readily available to anyone. However, one of the CSS concepts that many beginners find confusing is Positioning. Personally, it took me a long time to quite understand how to appropriately position different elements within a web-page using the Position Property, the reason is that you can easily get tricked that everything in your page is positioned correctly, until something goes wrong (for example an image is not where it is supposed to be), then you start tracing back through your code, making changes on the go, eventually things get messy!

Lets take a look at the different values that can be assigned to the position property:

  • Static (Default Value): The elements is positioned (rendered) in the same order as in the HTML code.
  • Absolute: The element is positioned relative to its first un-statically positioned parent (container) element. Otherwise, the element is positioned relative to the browser window.
  • Fixed: The element is positioned relative to the browser window. It remains in the same position even if the page is scrolled up or down.
  • Relative: The element is positioned relative to its own normal position.

There are 4 other important properties that assist in the positioning of an element on a page. These are (top, bottom, right & left). The way those 4 properties behave is dependent on the value of the position property.

Generally speaking, the way you control the position of an element on a page is through a combination of the property value along with one or more of the 4 properties mentioned above.
Now before you start going crazy with CSS, go through the examples below, and all of this will start making sense.

This is why you see them positioned in the exact same order they were placed in the HTML code, in other words they are all set in the normal flow of the document.

In this second example, you can see that the position of the 2 most inner boxes was changed to absolute. Also, you can see that the right property was also set to zero for both. Here we're faced with 3 questions:
  1. Why are both the boxes on top of each of other?
    By setting both of their positions to absolute, both boxes are repositioned in relation to the browser window, they no longer abide to the normal flow of the document. Thus, they fall over one another.
  2. Why did both boxes move all the way to the right of the screen?
    When we set the right property of an element, we're essentially setting the distance between the right edge of that element and its containing element (in this case the outer white box).
  3. But why, are they positioned in relation to the outer white box and not their container (the gray box)?
    The reason is that the position of the gray container box is static. If you want to see what happens if we change the position value of the gray container box, check the next example!

In this third example, the gray container has the position set to relative.
Now you can see that both the inner boxes are positioned at the upper right corner of th gray container. The reason is that since the gray container now has a position value other than static, it can serve as the container for all inner elements.

In this fourth example you see that the position of both the inner boxes is set to relative. And we can also see that the green box is given a left value of 5em. Now this raises a few questions:
  1. Why did both boxes move back into the gray container box?
    Because of their (relative) position. In this case the element is positioned relative to its normal position in the document.
  2. Why is the Yellow box not positioned to the right of the gray container even though it has a right value of 0?
    Since the yellow box has a relative position, it was shifted 0 units to the right of its initial (normal) position. Since the value is 0, it didn't move at all!
  3. Why is the green box shifted from the left?
    Again, since the green box has a relative position, and a left value of 5em, it was pushed to the left 5 units away from its initial (normal) position

Here are a few final notes:

  • Fixed position is same as absolute position, except that it sticks to its position on the screen, even if you scroll up or down.
  • If you have both left and right properties set, the left property will override and the right property won't work. The same happens with bottom and top (top has priority)
  • If the element is absolutely positioned, it will stretch its width to contain all the content within it. Unlike the relative position, which will always to stretch to 100% of the space that it can fill.
